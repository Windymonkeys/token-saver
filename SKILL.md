# Token Saver - Token 节省技能

## 触发条件

**文言文模式触发词**：
- "用文言文说..."
- "翻译成文言"
- "压缩这段话"
- "简短一点"
- "节省 token"

**结构化模式触发词**：
- "用 JSON 格式"
- "给我一个模板"
- "格式化输出"
- "结构化表示"
- "生成 schema"

---

## 两种模式

### 模式一：文言文压缩

将冗长的现代汉语转换为简洁的文言表达。

#### 词库位置
`references/wenyan_mapping.json`（450+ 词条）

#### 转换规则

1. **优先单字**：优先使用单字映射（如"吾"而非"吾等"）
2. **语境适配**：根据上下文选择最合适的文言词汇
3. **保留原意**：翻译后必须确保语义不变，必要时保留原词
4. **不造词**：不存在的文言词汇直接保留原词
5. **专业术语保留**：技术术语（API、代码、框架名）不翻译

#### 示例

| 现代汉语 | 文言文 | 节省 |
|---------|--------|------|
| 你知道这个方法吗？ | 汝知此法乎？ | 3字 |
| 我认为这个结果很好 | 吾以为此果甚善 | 2字 |
| 因为他很努力，所以成功了 | 因彼甚勤，故成矣 | 4字 |
| 如果你有时间的话，可以帮助我吗？ | 若汝有时，可助吾乎？ | 4字 |

#### 自动化脚本

可使用 `scripts/translate.py` 进行自动转换：

```bash
# 直接转换
python translate.py "你要转换的文本"

# 显示统计
python translate.py "文本" --stats

# 文件转换
python translate.py --file input.txt --output output.txt

# 运行测试
python translate.py --test
```

---

### 模式二：结构化输出

使用预定义模板替代自然语言描述。

#### 模板位置
`references/struct_templates.md`

#### 常用模板速查

| 用途 | 模板 |
|------|------|
| API 响应 | `{code, message, data}` |
| 分页数据 | `{items, pagination}` |
| 错误处理 | `{code, message, data: null}` |
| 函数注释 | `/** @param @returns */` |
| Markdown 表格 | `\| 列1 \| 列2 \|` |
| 测试用例 | `{id, input, expected}` |

## 使用决策树

```
用户输入 → 包含触发词？
    │
    ├── 文言/古文/压缩 → 执行文言文压缩
    │
    ├── JSON/模板/格式 → 执行结构化输出
    │
    └── 无触发词 → 正常对话
```

## 效果分析

| 文本长度 | 平均节省率 |
|----------|-----------|
| 短文本（<50字） | 10-20% |
| 中等文本（50-200字） | 15-30% |
| 长文本（>200字） | 20-40% |

**最佳使用场景**：
- 背景信息描述
- 任务需求说明
- 多轮对话历史摘要
- 重复性内容压缩

## 注意事项

1. **不强制翻译**：用户未明确要求时，不主动翻译
2. **确保可逆性**：重要信息确保 AI 和用户都能正确理解
3. **混合使用**：两种模式可以结合
4. **保留关键信息**：专业术语、数字、代码保留原样
