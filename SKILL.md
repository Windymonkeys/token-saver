# Token Saver - Token 节省技能

## Overview

本技能通过两种方式帮助节省 Token：

1. **文言文压缩**：将冗长的现代汉语转换为简洁的文言表达
2. **结构化输出**：使用预定义模板替代自然语言描述

## 两种模式

### 模式一：文言文压缩

当用户要求"用文言文说"、"翻译成文言"、"简短一点"时，执行以下流程：

#### 映射规则来源
词库位于 `references/wenyan_mapping.json`，包含：
- 常见代词（我→吾、你→汝/子、这个→此）
- 常用动词（知道→知、认为→以为、可以→可）
- 常见形容词（好的→善、重要的→重、正确的→正）
- 常用名词（人→人/民、时间→光阴/辰、地方→处/所）
- 连接词与虚词（但是→然、如果→若、因为→因/盖）
- 常用句式（我认为→吾以为、你知道吗→汝知之乎）
- 现代网络语（点赞→赞、内卷→竞争激烈、躺平→息事）

#### 转换规则

1. **优先单字**：优先使用单字映射（如"吾"而非"吾等"）
2. **语境适配**：根据上下文选择最合适的文言词汇
3. **保留原意**：翻译后必须确保语义不变，必要时保留原词
4. **不造词**：不存在的文言词汇直接保留原词
5. **技术术语**：专业术语（如 API、代码）一般不翻译

#### 示例

| 现代汉语 | 文言文 |
|---------|--------|
| 你知道这个方法吗？ | 汝知此法乎？ |
| 我认为这个结果很好 | 吾以为此果甚善 |
| 因为他很努力，所以成功了 | 彼甚勤，故成 |
| 这个问题很难解决 | 此题难解 |
| 谢谢你告诉我这个消息 | 多谢汝告吾此讯 |

#### 执行方式

**方式 A - 直接转换**（简单情况）：
直接用文言词汇替换，保持原句结构。

**方式 B - 使用词库**（推荐）：
1. 读取 `references/wenyan_mapping.json`
2. 查找对应词汇
3. 按规则组合成句

**方式 C - 重新表达**（复杂情况）：
完全用文言思维重新表达，参考：
- 主谓宾倒装为 主宾谓
- 多重修饰简化
- 保留核心信息和意图

---

### 模式二：结构化输出

当用户要求"用 JSON"、"格式输出"、"给我一个模板"时，直接使用 `references/struct_templates.md` 中的模板。

#### 常用模板速查

| 用途 | 模板 |
|------|------|
| API 响应 | `{code, message, data}` |
| 分页数据 | `{items, pagination: {page, pageSize, total}}` |
| 错误处理 | `{code, message, data: null}` |
| 数据定义 | `{field: {type, description, required}}` |
| 代码注释 | `/** @param @returns @throws */` |
| Markdown 表格 | `\| 列1 \| 列2 \|` |

## 使用决策树

```
用户输入 → 是否为长文本（>200字）？
    │
    ├── 是 → 用户要求压缩？
    │       │
    │       ├── 文言/古文 → 执行文言文压缩
    │       ├── 格式/模板 → 执行结构化输出
    │       └── 默认 → 询问用户偏好
    │
    └── 否 → 是否为复杂任务？
            │
            ├── 是 → 建议使用结构化模板
            └── 否 → 正常对话
```

## 注意事项

1. **不强制翻译**：用户未明确要求时，不主动翻译
2. **可逆性**：重要信息确保 AI 和用户都能正确理解
3. **混合使用**：两种模式可以结合（如文言文+结构化）
4. **反馈机制**：如翻译不准确，标记并跳过该词
